## What I learned

- The cascade decides which rule wins — specificity order is element → class → ID → inline, and when specificity ties, the rule written later wins
- Setting defaults on `body` (like `font-family`) lets everything inherit from it, so you only override what needs to be different
- External CSS is better than internal or inline because all your pages can share one file and stay consistent
- DevTools is actually useful — you can see which rules are being overridden (they show up crossed out) and test changes live without touching your code
- When prompting AI for CSS, being vague gets you generic output. Naming specific colors, fonts, and sizes gets you something actually usable

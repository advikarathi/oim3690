## What I learned

- Arrays hold many values in one variable, indexed from zero — `fruits[0]` is the first item, not `fruits[1]`.
- Three loop types cover most needs: `for...of` when you just need each value, the classic `for` when you specifically need the index, and `forEach` when you want both at once — useful for things like wiring up the same click handler across several elements.
- The pattern behind most real apps — a feed, a search box, a gallery — is just an array, a loop, and an `if`. Once that clicks, rendering a list of API results is the same pattern as anything else that loops over data.
- A loop needs a correct start, stop, and step — we saw a bad one freeze the browser tab live in class, which was a good reminder to double-check loop conditions before running anything, especially AI-written ones.
- `fetch()` only resolves with a `Response` object, not usable data — you need a second `await response.json()` before there's anything to actually loop over:
  ```js
  const response = await fetch(API_URL);
  const data = await response.json(); // now data.meals is usable
  ```
- TheMealDB's `filter.php?a=<area>` only returns summary fields (`idMeal`, `strMeal`, `strMealThumb`, `strArea`, `strCountry`) — getting full ingredients and instructions for a recipe needs a second call to `lookup.php?i=<id>`.
- The area list (`list.php?a=list`) now covers 195+ countries, but the actual recipe database is much smaller (IDs only run into the low 53,000s), so a lot of those areas will return zero meals — that's a real case to design around, not a bug.

## What I learned

### JS can grab HTML elements and mess with them
- `document.querySelector("#id")` and `document.getElementById("id")` both let you pull an element out of the page so you can do stuff to it
- once you have it, you can change styles, text, classes — basically anything

### Event listeners
- instead of putting `onclick=` directly in the HTML, you can attach behavior in the JS with `.addEventListener()` — feels cleaner, keeps things separated
- didn't realize you could listen for `"mouseover"` too, not just clicks — that's what makes the mailbox grow when you hover

### How the random color thing works
- there's an array of hex colors and it picks one randomly using `Math.floor(Math.random() * n)` — `Math.random()` gives you a decimal between 0 and 1, then `Math.floor` rounds it down so it lands on a real index
- makes sense once I saw it written out

### The hide/seek toggle uses CSS, not JS directly
- the button doesn't actually hide the text itself — it just adds or removes a class called `hidden`, and the CSS is what does `.hidden { display: none; }`
- kind of a weird split of responsibility but I get why

### `parseInt` with a fallback
- the mailbox font size is read with `parseInt(mailbox.style.fontSize) || 30` — the `|| 30` is there because if the size hasn't been set yet, `parseInt` would return `NaN` and the whole thing would break

### Browser-only vs. API-dependent tools (MP2 vs. MP3 distinction)
- in class we talked through what makes something a good MP2 scope: has to work entirely in the browser with no live API calls — just HTML, CSS, JS
- if it needs real-time data from an external service (weather, prices, etc.) it's an MP3-level idea
- useful filter: ask "does this need a server to be useful?" — if yes, it's MP3

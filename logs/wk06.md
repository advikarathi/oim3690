## What I learned

### How the script connects to the page

The `<script src="script.js">` tag sits at the bottom of `index.html`, right before `</body>`. Putting it there means all the HTML elements already exist on the page before JavaScript tries to find them. If it were in the `<head>`, the script would run before the elements load and nothing would work.

### `getElementById`

The app uses `getElementById` to grab specific HTML elements by their `id`. Each one targets a single element:

- `todo-form` — the form where you type a task
- `task-input` — the text input field
- `due-date-input` — the date picker
- `todo-list` — the `<ul>` where task items go
- `task-summary` — the "X tasks left" counter
- `empty-state` — the "Nothing here yet" message
- `clear-completed` — the button that removes done tasks

### Event listeners

The app listens for three types of events:

- `submit` on the form — fires when you click **Add task** or press Enter. It stops the page from reloading, builds a task object, clears the form, and redraws the list.
- `change` on each checkbox — fires when you check or uncheck a task. It flips that task's `done` value and redraws the list.
- `click` on each Delete button — removes that task and redraws.
- `click` on **Clear completed** — keeps only tasks that aren't done yet and redraws.

### How JavaScript updates the page

Instead of editing individual elements, the app rebuilds the whole list every time something changes via `updatePage()`:

- `taskList.innerHTML = ''` wipes the current list
- `document.createElement(...)` builds new list items, checkboxes, text, date labels, and buttons from scratch
- `textContent` puts the actual content into those elements
- `className` assigns CSS classes like `task-details`, `due-date`, and `delete`
- `classList.toggle('completed', task.done)` adds or removes the strikethrough style based on whether a task is done
- `append` / `appendChild` places everything into the list
- `emptyState.style.display` shows or hides the "Nothing here yet" message
- `form.reset()` clears both inputs after adding a task; `taskInput.focus()` puts the cursor back so you can type the next task right away

### `let`, `const`, and `function`

- `const` is used for things that always point to the same element or function. The values can still change (like `checkbox.checked` updating), but the variable never points somewhere new.
- `let tasks = []` starts as an empty array and gets fully replaced when tasks are deleted or cleared — that's why it uses `let` instead of `const`.
- `function updatePage()` wraps all the page-redrawing logic in one place. Every event listener just calls it at the end instead of manually updating the list, counter, and empty state separately.

### Converter and discount practice

- In `converter.html`, the main function is `toFahrenheit(celsius)`. It returns `celsius * 9 / 5 + 32`, so `100` becomes `212`.
- The converter uses `Number(celsiusInput.value)` and `Number(fahrenheitInput.value)` because input values start as text, even when the input type is `number`.
- I enhanced the converter into a two-way converter, so typing in Celsius updates Fahrenheit and typing in Fahrenheit updates Celsius.
- In `discount.html`, the main function is `getDiscount(total, isMember)`. It uses `if`, `else if`, `&&`, and `||`.
- The discount logic is: total `>= 100` and member gives `20%`; total `>= 100` or member gives `10%`; otherwise the discount is `0%`.
- The checkout page then uses that returned percent to show the discount amount and the final price.

### New storage code explained

- `const STORAGE_KEY = 'fresh-list-items'` gives the saved list a name in the browser’s local storage, keeping it separate from data other sites save.
- `let tasks = loadTasks()` runs when the page opens, so the app starts with saved items instead of always starting with an empty array.
- `localStorage.getItem(STORAGE_KEY)` gets saved text from the browser. If there is no saved list yet, the function returns `[]`, an empty array.
- `JSON.parse(savedTasks)` turns the saved text back into a JavaScript array of grocery items. `try` and `catch` mean broken saved data will not crash the app; it uses an empty list instead.
- `JSON.stringify(tasks)` turns the JavaScript array into text, which `localStorage.setItem(...)` can save.
- `saveTasks()` runs in `updatePage()`, so adding, checking, deleting, or clearing an item saves the newest version for the next refresh.

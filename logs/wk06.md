# Week 6 Log

## Reading `todo/index.html`

### 1. Where is the script?

The `<script>` tag is near the bottom of `todo/index.html`, just before `</body>`. The JavaScript is inside the HTML file, not in its own `.js` file. Putting it at the bottom means the HTML elements are already on the page when the JavaScript tries to find them.

### 2. `getElementById` and `querySelector`

This app uses `getElementById` (and no `querySelector`). Each one selects one HTML element by its `id`:

- `todo-form` selects the form where a task is entered.
- `task-input` selects the text box for the task name.
- `due-date-input` selects the date picker.
- `todo-list` selects the empty `<ul>` where task `<li>` items are placed.
- `task-summary` selects the text that says how many tasks are left.
- `empty-state` selects the “Nothing here yet” message.
- `clear-completed` selects the button that removes completed tasks.

### 3. Event listeners

- The form listens for `submit`. This happens when I click **Add task** or press Enter. It stops the page from reloading, creates a task object, clears the form, and redraws the page.
- Every checkbox created for a task listens for `change`. This happens when it is checked or unchecked; it changes that task’s `done` value and redraws the page.
- Every Delete button listens for `click`. It removes its own task and redraws the page.
- The **Clear completed** button listens for `click`. It keeps only tasks that are not done and redraws the page.

### 4. Where JavaScript changes the page

- `taskList.innerHTML = ''` clears the visible task list before rebuilding it.
- `document.createElement(...)` creates new list items, checkboxes, text, dates, and buttons.
- `textContent` puts task names, due dates, button labels, and the “tasks left” message onto the page.
- `className` gives created elements CSS classes such as `task-details`, `due-date`, and `delete`.
- `classList.toggle('completed', task.done)` adds or removes the `completed` class, which makes finished tasks faded and crossed out.
- `append` and `appendChild` put the new elements into the list item and then onto the page.
- `emptyState.style.display` shows the empty message when there are no tasks and hides it when tasks exist.
- `form.reset()` clears the form after a task is added, and `taskInput.focus()` puts the cursor back into the task box.

### 5. `let`, `const`, and `function`

- `const` is used for things that keep referring to the same thing, like the form, inputs, buttons, and functions. The contents can sometimes still change: for example, `checkbox.checked` changes even though the `checkbox` variable is `const`.
- `let tasks = []` makes a variable that can change. It starts as an empty array, then stores every task. It uses `let` because the program sometimes replaces the whole array after deleting or clearing tasks.
- `function updatePage()` names a reusable set of instructions. It redraws the list, summary, and empty message every time task information changes. I think functions keep repeated code in one place: instead of separately updating the page after every possible action, the app can just call `updatePage()`.
